• Requerimientos Funcionales se centran en qué debe hacer el sistema, es decir, las funciones que debe realizar.
  • Requerimientos No Funcionales se centran en cómo el sistema debe funcionar y bajo qué condiciones, es decir, las características cualitativas del sistema.
Comparación

Funcionalidad

Los requerimientos no funcionales describen cómo debe funcionar el sistema. No se centran en las funcionalidades que el sistema proporciona, sino en las cualidades y restricciones bajo las cuales esas funcionalidades deben operar.

Características:

  • Calidad del Sistema: Se enfocan en atributos como el rendimiento, la seguridad, la usabilidad, la escalabilidad, la mantenibilidad, entre otros.
  • Restricciones: También pueden incluir limitaciones técnicas, estándares de cumplimiento, y restricciones operacionales.
  • Ejemplos:
    • El sistema debe poder manejar 1000 transacciones por segundo.
    • El tiempo de respuesta del sistema no debe exceder los 2 segundos bajo carga normal.
    • El sistema debe cumplir con las regulaciones GDPR para la protección de datos.
No funcionales​

Los requerimientos funcionales describen lo que el sistema debe hacer.

Se refieren a las funcionalidades y servicios específicos que el sistema debe proporcionar, detallando cómo debe comportarse en respuesta a ciertas entradas o situaciones.

Características:

  • Especificidad: Son específicos y detallados, describiendo acciones, procesos y tareas que el sistema debe realizar.
  • Orientados al Usuario: Están enfocados en las necesidades del usuario final y cómo el sistema debe satisfacer esas necesidades.
  • Ejemplos:
    • El sistema debe permitir a los usuarios iniciar sesión utilizando un nombre de usuario y contraseña.
    • El sistema debe enviar una notificación por correo electrónico cuando se complete una transacción.
    • El sistema debe permitir a los usuarios buscar productos por categoría.
